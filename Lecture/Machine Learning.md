# Машинное обучение (Machine Learning)  
  
Машинное обучение — раздел искусственного интеллекта, в котором алгоритмы обучаются находить закономерности в данных и делать предсказания без явного программирования правил.  
  
Главная идея ML — построить функцию:  
  
f(x) → y  
  
где:  
  
- x — входные данные (признаки)  
- y — целевая переменная  
  
Модель пытается приблизить неизвестную зависимость между ними.  
  
---  
  
# Типы задач машинного обучения  
  
## Классификация  
  
Классификация — задача определения класса объекта.  
  
Примеры:  
  
- спам / не спам  
- болезнь / нет болезни  
- тип изображения  
  
Типы:  
  
**Бинарная классификация**  
  
2 класса.  
  
Пример:  
  
- спам / не спам  
  
**Многоклассовая классификация**  
  
Больше двух классов.  
  
Пример:  
  
- классификация изображений животных.  
  
---  
  
## Регрессия  
  
Регрессия — задача предсказания **числового значения**.  
  
Примеры:  
  
- цена квартиры  
- температура  
- спрос на товар  
  
Модель пытается предсказать непрерывную величину.  
  
---  
  
## Кластеризация  
  
Кластеризация — задача группировки объектов без заранее известных классов.  
  
Алгоритм объединяет объекты в группы на основе схожести.  
  
Примеры алгоритмов:  
  
- K-Means  
- DBSCAN  
- Hierarchical Clustering  
  
Применение:  
  
- сегментация клиентов  
- анализ поведения пользователей  
  
---  
  
## Снижение размерности  
  
Методы, которые уменьшают количество признаков, сохраняя основную информацию.  
  
Зачем это нужно:  
  
- ускорение обучения  
- уменьшение шума  
- визуализация данных  
  
Популярные методы:  
  
- PCA (Principal Component Analysis)  
- t-SNE  
- UMAP  
  
---  
  
# Переобучение и недообучение  
  
## Переобучение (Overfitting)  
  
Модель слишком хорошо запоминает обучающие данные и плохо работает на новых данных.  
  
Признаки:  
  
- высокая точность на обучении  
- низкая точность на тесте  
  
Причины:  
  
- слишком сложная модель  
- мало данных  
- слишком много признаков  
  
Методы борьбы:  
  
- регуляризация  
- увеличение датасета  
- уменьшение сложности модели  
- кросс-валидация  
  
---  
  
## Недообучение (Underfitting)  
  
Модель слишком простая и не может уловить закономерности.  
  
Признаки:  
  
- плохие результаты и на обучении, и на тесте.  
  
Причины:  
  
- слишком простая модель  
- недостаточное обучение  
- неправильные признаки  
  
---  
  
# Bias-Variance Tradeoff  
  
В машинном обучении существует компромисс между:  
  
**Bias (смещение)** — ошибка из-за слишком простых предположений модели.  
  
**Variance (дисперсия)** — ошибка из-за чувствительности модели к данным.  
  
Простые модели:  
  
- высокий bias  
- низкий variance  
  
Сложные модели:  
  
- низкий bias  
- высокий variance  
  
Цель — найти баланс.

# Метрики качества моделей  
  
Метрики используются для оценки качества модели.  
  
## Метрики классификации  
  
### Accuracy  
  
Доля правильных предсказаний.  
  
Accuracy = (TP + TN) / (TP + TN + FP + FN)  
  
Недостаток:  
  
плохо работает при несбалансированных данных.  
  
---  
  
### Precision  
  
Доля правильных положительных предсказаний.  
  
Precision = TP / (TP + FP)  
  
---  
  
### Recall  
  
Доля найденных положительных объектов.  
  
Recall = TP / (TP + FN)  
  
---  
  
### F1 Score  
  
Гармоническое среднее Precision и Recall.  
  
F1 = 2 * (Precision * Recall) / (Precision + Recall)  
  
---  
  
### ROC-AUC  
  
Метрика качества бинарной классификации.  
  
Показывает способность модели разделять классы.  
  
---  
  
## Метрики регрессии  
  
### MAE (Mean Absolute Error)  
  
Средняя абсолютная ошибка.  
  
MAE = mean(|y - y_pred|)  
  
---  
  
### MSE (Mean Squared Error)  
  
Средняя квадратичная ошибка.  
  
MSE = mean((y - y_pred)^2)  
  
---  
  
### RMSE  
  
Корень из MSE.  
  
---  
  
### R² (коэффициент детерминации)  
  
Показывает, какую долю дисперсии объясняет модель.  
  
---  
  
# Гиперпараметры моделей  
  
Гиперпараметры — параметры, которые задаются **до начала обучения**.  
  
Примеры:  
  
- learning rate  
- глубина дерева  
- количество деревьев  
- размер батча  
- количество эпох  
  
Подбор гиперпараметров называется **Hyperparameter Tuning**.  
  
Методы:  
  
- Grid Search  
- Random Search  
- Bayesian Optimization  
  
---  
  
# Batch, Epoch и Iteration  
  
## Batch  
  
Batch — часть данных, используемая за один шаг обучения.  
  
---  
  
## Epoch  
  
Epoch — один полный проход по всему обучающему датасету.  
  
---  
  
## Iteration  
  
Iteration — одно обновление весов модели.  
  
Количество итераций зависит от размера batch.  
  
---  
  
# Feature Engineering  
  
Feature Engineering — процесс создания новых признаков.  
  
Примеры:  
  
- извлечение года из даты  
- агрегирование данных  
- создание взаимодействий признаков  
- нормализация  
  
Хорошие признаки часто важнее выбора модели.  
  
---  
  
# Feature Selection  
  
Feature Selection — выбор наиболее важных признаков.  
  
Методы:  
  
- корреляционный анализ  
- feature importance  
- L1 регуляризация  
- Recursive Feature Elimination  
  
Цель:  
  
- уменьшить размерность  
- улучшить качество модели  
- ускорить обучение  
  
---  
  
# Pipeline в машинном обучении  
  
Pipeline — последовательность этапов обработки данных и обучения модели.  
  
Типичный pipeline:  
  
1. Предобработка данных  
2. Feature Engineering  
3. Масштабирование  
4. Обучение модели  
5. Валидация  
6. Тестирование  

Pipeline помогает избежать утечек данных (data leakage).
